### Core Capabilities

#### 💸 Micropayment Channels
- **Pooled Settlement**: Batch thousands of micro-transactions into single settlements
- **Sub-Cent Precision**: Handle payments down to $0.000001 (one-millionth of a dollar)
- **Channel Balancing**: Automatic liquidity management across payment channels
- **Zero-Marginal-Cost**: Once channel is open, transactions are essentially free

**Example**: Agent makes 10,000 API calls at $0.001 each = $10 total. Instead of 10,000 payments, one $10 settlement with detailed line-item accounting.

#### 🌊 Streaming Payments
- **Per-Second Billing**: Pay for compute as it runs, not after
- **Automatic Cutoffs**: Hit budget limit? Stream stops instantly
- **Proportional Charging**: Use 3.7 seconds of GPU? Pay for exactly 3.7 seconds
- **Bi-Directional Streams**: Agents can simultaneously pay and receive

```javascript
// Stream payment example
const stream = await agentpay.createStream({
  recipient: 'gpu-provider.agentpay.id',
  rate: '$0.0001/second',  // $0.36/hour
  maxTotal: '$5.00',
  purpose: 'inference-job-12847'
});

// Payment flows automatically while compute runs
await runInference(model, data);
stream.close();  // Settles instantly
```

### How It Works

#### 1. Channel Creation
When agents need to transact frequently, they open a payment channel:
- Deposit funds into smart escrow
- Unlimited off-chain transactions
- Cryptographic receipts for each transaction
- Settle on-chain/to-bank periodically

#### 2. Transaction Flow
```
Agent A wants to pay Agent B $0.003:
1. A signs payment intent with cryptographic proof
2. AgentPay validates against A's policies
3. Channel balance updated (no bank involved yet)
4. B receives instant confirmation
5. B provides service
6. Transaction logged for settlement batch
```

#### 3. Settlement Aggregation
```
End of settlement period (configurable: 1hr to 1 day):
- 50,000 transactions between A and B
- Net amount: A owes B $127.43
- Single bank transfer executes
- All 50,000 transactions have full audit trail
```
