How It Works

Three Steps to Zero-Storage Verification

1

User Encrypts Locally

Identity documents and data are encrypted and stored only on the user's device. Keys are protected by hardware security.

2

Generate ZK Proof

The user's device generates a zero-knowledge proof that demonstrates the required attributes without revealing raw data.

3

Verify Cryptographically

ProofID verifies the mathematical proof and returns a simple True/False to the business. No data ever stored.

Developer Experience

Simple API. Zero Data.

verification.js 
// ProofID: Verify age without receiving DOB
const proofid = require('@proofid/sdk');

const result = await proofid.verify({
  userId: 'user_abc123',
  claims: [
    { type: 'age', operator: 'gte', value: 21 },
    { type: 'country', operator: 'eq', value: 'US' },
    { type: 'watchlist', operator: 'not_in', value: 'OFAC' }
  ]
});
